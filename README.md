# pc_sales
**PC Sales Database Analysis**    This SQL project analyzes a comprehensive dataset of computer hardware sales across multiple shops and regions. Using advanced queries, joins, aggregations, and window functions, it uncovers key insights into product performance, regional trends, customer behavior, profitability margins, repair costs, etc
his repository contains SQL scripts, queries, and documentation for analyzing a **PC / computer hardware sales database**.

Main objectives:
- Understand sales patterns by shop, region, product type, and time
- Identify high-value customers and credit risks
- Evaluate shop performance (age vs sales, repairs cost)
- Analyze profitability (discounts, finance, margins)
- Support business decisions like inventory, marketing, and staffing

Great for data analysts, SQL learners, or anyone building a portfolio project!

## Dataset Description

The dataset simulates real-world PC retail sales across multiple shops and regions.

**Key entities/tables (inferred from columns):**
- Shops / Stores
- PCs / Products (Make, Model, Type, Storage, RAM)
- Customers
- Sales Transactions
- Repairs / After-sales
- Finance & Shipping

Sample fields include:
- Geographic: Continent, Country/State, Province/City
- Product: PC_Make, PC_Model_Type, Storage_Type, RAM
- Sales: Cost_Price, Sale_Price, Discount_Amount, Sale_Price, Payment_Method, Purchase_Date
- Customer: Customer_Name, Surname, Contact_Number, Email, Credit_Score
- Other: Shop_Age, Priority, Cost_of_Repairs, Total_Sales_per_Employee

(No real sensitive data — likely anonymized or synthetic.)
